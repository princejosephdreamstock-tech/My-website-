AFP-ERL-001 · Meridian Evidence Request List
ERL-01
AI System InventoryA list of all AI systems currently in production use within your organisation. Include vendor name, system function, date deployed, and the business unit responsible. If no formal inventory exists, provide a written description of known AI tools in use.
ERL-02
Article 14(4) Oversight DesignationThe name and role of the individual formally designated as the Article 14(4) oversight authority for each AI system. If no such designation exists, state this explicitly. Do not substitute a team name, department, or shared inbox.
ERL-03
Decision Log Sample — 90 DaysAn export of AI decision log data covering the last 90 days. Must include: decision ID, decision timestamp, decision outcome, and the system or user account ID logged against each decision. Anonymise personal data before submission. Format: CSV or JSON.
ERL-04
Review Log Sample — 90 DaysAn export of human review log data covering the same 90-day period as ERL-03. Must include: review ID, decision ID (shared reference to ERL-03 where it exists), reviewer account ID, review timestamp, review outcome, and any rationale field populated by the reviewer. Format: CSV or JSON.
ERL-05
Annex IV Technical DocumentationYour current Annex IV technical documentation if it exists. If not yet produced, confirm this in writing. The absence of Annex IV documentation is a finding in itself and will be documented accordingly.
ERL-06
Override and Escalation PolicyAny written policy, procedure, or protocol governing how human reviewers are expected to override, escalate, or halt AI decisions. Internal documents, draft policies, and email instructions are all acceptable. If no written policy exists, state this explicitly.
ERL-07
Post-Market Monitoring PlanAny documentation of your post-market monitoring framework for AI systems in scope. If no formal plan exists, confirm this in writing. Absence of a post-market monitoring plan is an Article 72 finding.
ERL-08
Vendor Data Processing AgreementsThe DPA or AI-specific data processing agreement from each AI vendor whose system is in scope. If these have not been reviewed for AI Act alignment, confirm this. Unreviewed vendor agreements are a common source of material findings.
